Sebuah platform streaming lokal, "Nusantara Flix," memiliki sistem untuk mengelola katalog film dan series mereka. Bagian Konten sering perlu mengoreksi atau memperbarui detail media yang sudah tayang, seperti tahun rilis, genre, atau rating baru.
Anda diminta untuk membuat dan menguji endpoint API PUT yang menangani pembaruan data media pada tabel media di database MySQL, dengan struktur yang sangat mirip dengan contoh Update yang ada di Modul Pertemuan 5.
Asumsi:
1. memiliki file skrip Node.js utama dengan konfigurasi server (app) dan koneksi database (pool).
2. memiliki tabel di database MySQL bernama media dengan kolom minimal: id_media, judul, tahun_rilis, dan genre.
3. Endpoint ini akan menggunakan HTTP PUT dan menerima data baru dari req.body.

Soal Proyek (Tugas Implementasi)
Tambahkan endpoint baru pada skrip Node.js Anda untuk mengimplementasikan fungsi Update Data Media yang memenuhi kriteria berikut:

1. Implementasi Endpoint PUT
Tambahkan endpoint app.put('/media/:id_media") pada file server Anda.
2. Pengambilan Data dan Parameter
Akses:
• ID Media (id_media) yang akan diperbarui dari req.params. Data baru (judul, tahun_rilis, dan genre) dari req.body.
3. Query Pembaruan Database (UPDATE)
Gunakan objek koneksi database (pool) untuk menjalankan query MySQL. Tulis query UPDATE yang akan memperbarui kolom judul, tahun_rilis, dan genre pada tabel media berdasarkan id_media yang diberikan, seperti yang dicontohkan pada skrip UPDATE di modu Pertemuan 5.
4. Penanganan Respons yang Sesuai
Terapkan logic penanganan respons:
• Sukses: Jika query berhasil dan results.affectedRows > 0, kirim respons status 200 OK dengan pesan seperti "Data media berhasil diperbarui".
• Tidak Ditemukan: Jika query berhasil tetapi results.affectedRows adalah 0, kirim respons status 404 Not Found dengan pesan seperti "Media tidak ditemukan".
Error Server: Tangani error yang mungkin terjadi selama eksekusi query.
